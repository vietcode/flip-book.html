The Making Of

Prepare the markups

We start by writing the basic markup for the book. We use <section> to represent a single book page within the component.

In browsers with Javascript disabled, the content is simply displayed linear. Javascript will transform the content into a flippable book.

We then use some styles to make the content looks nicer, using boom.css (http://alistapart.com/article/boom)

Modularity

Instead of writing everything in a single flip-book.html, we separate the three logical components (HTML, CSS, JS) and only merge them at compile time.

First thing we did is create a `src` folder with separate subfolders for logic (JS files, or even CoffeeScript files), styles (less files) and template (jade file). The choices are just personal, with the advantage of being less code, and more error checking at compile-time.

The template file allow us to inline styles and scripts from separate external files, which are the compiled from the corresponding pre-processor source files.

Using Gulp to build the element

Gulp keeps the build process under control. And in our structure, to build the elements we need Gulp to follow the following steps:

  • Compile the element's .less files to .css
  • Compile the element's .jade file to .html, embedding the .css and .js files.
